# 命令执行器 (Command Executor)

一个基于 Python 的本地 GUI 应用程序，用于管理并执行一系列 Windows 命令行任务。程序采用表格形式展示任务列表，支持实时输出、单任务控制、进程树强制终止等功能，适合需要批量执行命令或监控命令输出的场景。

## ✨ 功能特点

- **任务管理**：支持添加、修改、删除、查询任务。每个任务包含：
  - 任务名称
  - 说明（备注）
  - 多条命令行（按顺序执行）
- **实时输出**：每个任务拥有独立的输出标签页，执行过程中实时显示标准输出和错误输出，模拟命令行窗口效果。
- **单任务执行与停止**：
  - 点击“执行”仅运行当前选中任务。
  - 点击“停止”可终止当前正在运行的任务（强制终止整个进程树，避免残留子进程）。
- **智能按钮状态**：
  - 无任务执行时，按钮状态由选中任务自身状态决定（未执行/已完成/出错/已停止 → 修改、删除、执行可用）。
  - 有任务执行时：
    - 选中正在执行的任务 → 仅“停止”可用。
    - 选中其他任务 → 根据该任务状态决定“修改”是否可用，“删除”禁用（防止索引错乱），“执行”禁用（不能同时运行两个任务）。
- **数据持久化**：任务列表自动保存到程序所在目录的 `data.json` 文件中，下次启动自动加载。
- **查询过滤**：输入关键词可快速过滤任务（匹配任务名称、说明或命令内容）。
- **窗口居中**：主窗口和所有对话框启动时自动居中。
- **完全隐藏命令行窗口**：执行命令时不会弹出任何控制台窗口，输出通过管道捕获显示在程序内部。

## 📦 安装与运行

### 环境要求
- 操作系统：Windows 7/8/10/11
- Python 3.6 或更高版本（如果直接运行源码）

### 直接运行源码
1. 克隆或下载本项目。
2. 确保已安装 Python 并添加到环境变量。
3. 在项目目录打开命令行，执行：
   ```bash
   python command_executor.py
   ```
   *注：本程序仅使用 Python 标准库（`tkinter`, `subprocess`, `threading`, `json`, `os`, `sys`, `time`），无需额外安装依赖。*

### 打包为独立的 EXE 文件
如果你希望得到一个无需 Python 环境即可运行的 EXE 文件，可以使用 `PyInstaller` 打包：

1. 安装 PyInstaller：
   ```bash
   pip install pyinstaller
   ```
2. 在项目目录执行打包命令：
   ```bash
   pyinstaller --onefile --windowed --name 命令执行器 command_executor.py
   ```
3. 打包后的 EXE 文件位于 `dist` 文件夹，可复制到任何位置运行。

## 🚀 使用说明

1. **启动程序**：双击运行 `命令执行器.exe` 或执行 `python command_executor.py`。
2. **添加任务**：点击“添加”按钮，在弹出的对话框中填写：
   - **任务名称**：简短描述。
   - **说明**：可选备注。
   - **命令行**：每行一条命令，按顺序执行。
3. **修改/删除任务**：在表格中选中任务后，点击对应按钮（执行任务期间删除按钮禁用）。
4. **查询任务**：在顶部输入关键词，点击“查询”或按回车过滤任务；点击“清除”恢复显示全部。
5. **执行任务**：
   - 选中一个状态为“未执行”/“已完成”/“出错”/“已停止”的任务，点击“执行”。
   - 程序将在后台按顺序执行该任务的所有命令，输出实时显示在下方对应标签页中。
   - 执行过程中可点击“停止”终止当前任务（强制终止所有相关进程）。
6. **查看输出**：点击标签页标题切换不同任务的输出内容。

### 任务状态说明
- **未执行**：尚未运行过。
- **运行中**：正在执行。
- **已完成**：所有命令成功执行完毕（允许部分命令返回非零值）。
- **出错**：执行过程中发生异常或命令返回非零值（用户未停止）。
- **已停止**：用户手动终止。

## 📁 数据存储

所有任务数据保存在程序所在目录的 `data.json` 文件中，格式如下：
```json
[
  {
    "description": "任务名称",
    "commands": ["command1", "command2"],
    "note": "说明"
  }
]
```
程序启动时自动加载，修改任务后自动保存。状态（如“运行中”）不会被持久化，每次启动均重置为“未执行”。

## 🛠️ 技术栈

- **GUI 框架**：`tkinter`（内置） + `ttk` 主题组件
- **进程管理**：`subprocess` + 多线程读取输出
- **数据持久化**：`json`
- **线程安全**：使用 `after` 方法更新 UI
